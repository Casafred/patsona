# 详细交互流程与话术

本文档详细描述 patent-taxonomy-builder 的5步交互流程，包括每步的 AskUserQuestion 具体话术、AI推荐策略和异常处理。

---

## Step 1: 确定技术方向

### 1.1 询问技术方向

**AskUserQuestion 配置：**
- header: "技术方向"
- questions:
  - question: "请输入你要构建分类体系的技术方向"
    options: ["电池/储能", "半导体/芯片", "人工智能", "生物医药", "新能源", "自定义输入"]
  - question: "该方向的覆盖范围？"
    options: ["全领域（含上下游）", "仅核心技术", "仅应用层", "自定义范围"]

### 1.2 AI推荐一级分支

**推荐策略：** 基于技术常识，为常见方向提供预置推荐。

| 技术方向 | 推荐一级分支 |
|---------|------------|
| 电池/储能 | 锂离子电池、固态电池、钠离子电池、燃料电池、超级电容 |
| 半导体/芯片 | 设计、制造、封装测试、材料、设备 |
| 人工智能 | 机器学习、计算机视觉、NLP、语音、机器人 |
| 生物医药 | 小分子药、抗体药、基因治疗、医疗器械、诊断 |
| 新能源 | 光伏、风电、氢能、储能、智能电网 |

**AskUserQuestion 配置：**
- header: "一级分支"
- questions:
  - question: "以下推荐的一级分支，哪些需要保留？"
    options: [AI推荐的分支列表 + "全部保留" + "自定义补充"]
  - question: "是否需要添加自定义一级分支？"
    options: ["不需要", "我来输入"]

### 1.3 异常处理

- **用户输入模糊方向**（如"电子"）：AI追问具体细分（消费电子/电力电子/光电）
- **用户输入过宽方向**（如"全部"）：建议缩小范围，解释过宽的分类体系难以落地
- **用户输入极窄方向**（如"纽扣电池"）：建议适当扩展到"电池"或"微型电池"

---

## Step 2: 逐层构建分支树

### 2.1 二级分支构建

对每个一级分支逐一询问：

**AskUserQuestion 配置：**
- header: "二级细分"
- questions:
  - question: "「{一级分支名}」的二级细分方向？"
    options: [AI推荐2-4个 + "自定义输入"]
  - question: "这些二级分支是否需要调整？"
    options: ["确认，继续", "需要增删", "重新推荐"]

### 2.2 三级分支构建

对每个二级分支逐一询问：

**AskUserQuestion 配置：**
- header: "三级细分"
- questions:
  - question: "「{二级分支名}」的三级细分方向？"
    options: [AI推荐2-4个 + "自定义输入"]
  - question: "是否需要继续细分到四级？"
    options: ["三级足够", "需要四级细分"]

### 2.3 AI推荐策略

**二级推荐逻辑：**
- 锂离子电池 → 正极材料、负极材料、电解液、隔膜、电芯设计
- 正极材料 → 磷酸铁锂、三元材料、钴酸锂、锰酸锂
- 三元材料 → NCM、NCA、高镍三元、中镍三元

**推荐原则：**
1. 优先按技术原理/材料体系细分
2. 其次按应用场景/产品形态细分
3. 避免按IPC分类号直接搬运（IPC粒度不适合专利分类）
4. 每层3-6个分支为宜，超过8个建议合并

### 2.4 实时展示

每完成一个一级分支的构建，展示当前分类树：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🌲 当前分类树 [Step 2/5]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
电池
├── 锂离子电池 ✅
│   ├── 正极材料
│   │   ├── 磷酸铁锂
│   │   ├── 三元材料
│   │   └── 钴酸锂
│   ├── 负极材料
│   │   ├── 石墨
│   │   └── 硅基
│   └── 电解液
├── 固态电池 🔄 [正在构建]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 2.5 异常处理

- **分支过多**（>8个）：建议合并相似分支
- **分支过少**（1个）：提示是否需要细分，或该分支可能不需要独立
- **层级不均**：部分一级分支有三级，部分只有二级，提示用户确认
- **命名冲突**：不同层级出现相同名称，建议区分（如"正极材料-锂离子"vs"正极材料-钠离子"）

## Step 3: 定义末端分支判定标准

### 3.1 逐分支定义

对每个末端分支（最细粒度），依次询问三类信息：

**AskUserQuestion 配置（第一组）：**
- header: "判定标准"
- questions:
  - question: "「{分支名}」的关键区分特征是什么？"
    options: [AI推荐特征 + "自定义"]
  - question: "用于粗筛的关键词有哪些？"
    options: [AI推荐关键词列表 + "自定义补充"]

**AskUserQuestion 配置（第二组）：**
- header: "边界规则"
- questions:
  - question: "当专利同时涉及「{分支A}」和「{分支B}」时如何归类？"
    options: ["按主权利要求判定", "按创新点判定", "归入更具体的分支", "自定义规则"]
  - question: "是否需要排除规则（什么情况下不应归入此分支）？"
    options: ["不需要", "我来指定排除条件"]

### 3.2 AI推荐判定标准

**推荐策略：**

| 分支类型 | 推荐判定标准 | 推荐关键词 |
|---------|------------|-----------|
| 材料类 | 包含该材料化学式/缩写/全称 | 化学式、材料名、缩写 |
| 工艺类 | 包含该工艺步骤/参数/设备 | 工艺名、关键参数 |
| 应用类 | 包含该应用场景/产品类型 | 应用名、产品名 |
| 结构类 | 包含该结构特征/组件名称 | 结构名、组件名 |

**关键词推荐原则：**
1. must_have：2-3个核心关键词（必须包含至少一个）
2. any_of：3-5个相关关键词（包含任意一个即可）
3. exclude：1-2个排除关键词（包含则排除）

### 3.3 边界案例处理

**常见边界模式：**

| 模式 | 示例 | 推荐处理 |
|------|------|---------|
| 材料交叉 | 同时涉及LFP和NCM | 按主权利要求所述材料分类 |
| 工艺+材料 | 新材料+新工艺 | 按创新点核心分类 |
| 应用+技术 | 新应用场景+已知技术 | 按技术本质分类，应用作为辅助标注 |
| 上游+下游 | 材料+电池包 | 按专利核心贡献分类 |

### 3.4 异常处理

- **无法定义区分特征**：提示该分支可能与相邻分支合并
- **关键词高度重叠**：建议增加must_have关键词或调整分支边界
- **边界规则过于复杂**：建议简化，优先保证80%案例可判定

## Step 4: 导入已标注样本

### 4.1 确认是否有样本

**AskUserQuestion 配置：**
- header: "样本导入"
- questions:
  - question: "是否有已标注的专利样本？"
    options: ["有，Excel格式", "有，其他格式", "没有，跳过此步"]
  - question: "（如有）Excel文件路径？"
    options: ["我来输入路径"]

### 4.2 解析与匹配

**解析流程：**
1. 读取Excel文件
2. 验证列格式（专利号、标题、摘要、分类标签）
3. 对每条样本，按关键词规则自动匹配到分支
4. 生成匹配报告

**匹配报告格式：**
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 样本匹配报告
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
总样本数: 50
自动匹配成功: 42 (84%)
需人工确认: 5 (10%)
无法匹配: 3 (6%)

匹配详情:
✅ 磷酸铁锂: 12条 → 全部匹配
✅ 三元材料: 10条 → 全部匹配
⚠️ 固态电池: 8条 → 6条匹配, 2条需确认
❌ 钠离子电池: 5条 → 3条匹配, 2条无法匹配
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 4.3 未匹配样本处理

对每条未匹配样本：

**AskUserQuestion 配置：**
- header: "样本确认"
- questions:
  - question: "专利「{标题}」无法自动匹配，应归入哪个分支？"
    options: [候选分支列表 + "新增分支" + "跳过"]

### 4.4 异常处理

- **Excel格式不符**：提示预期格式，建议调整
- **分类标签与分支不对应**：提示标签映射建议
- **样本过少**（<5条/分支）：提示该分支规则可能不够可靠
- **样本过多**（>200条）：建议抽样验证

---

## Step 5: 生成规则文件并校验

### 5.1 生成YAML文件

**文件路径：** `patsona/rules/{技术方向id}/taxonomy.yaml`

**生成逻辑：**
1. 将分类树转换为YAML层级结构
2. 填入每层的keywords、classification_rules、boundary_rules
3. 填入sample_patents（如有）
4. 生成id：使用方向名拼音或英文缩写

### 5.2 校验检查项

| 检查项 | 级别 | 说明 |
|-------|------|------|
| 末端分支无判定标准 | ERROR | 必须修复 |
| 关键词重叠（must_have） | WARNING | 建议检查 |
| 分支无样本 | WARNING | 建议补充 |
| 层级深度>4 | INFO | 建议简化 |
| exclude关键词冲突 | WARNING | 可能误排除 |

### 5.3 最终确认

**AskUserQuestion 配置：**
- header: "最终确认"
- questions:
  - question: "分类体系和规则是否满意？"
    options: ["满意，生成文件", "需要调整部分内容", "需要大改"]
  - question: "是否需要导出分类树为独立文件？"
    options: ["不需要", "导出为Markdown", "导出为JSON"]

### 5.4 异常处理

- **校验有ERROR**：必须修复后才能生成文件
- **用户不满意**：返回对应Step进行调整
- **文件写入失败**：检查目录权限，提示用户手动创建目录
