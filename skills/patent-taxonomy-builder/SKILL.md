# 专利分类体系构建器

## 核心定位

通过多轮对话引导用户逐步构建专利分类体系，最终生成YAML规则文件到patsona项目的 `rules/` 目录。

**终极目标：** 产出结构完整、判定标准清晰的专利分类YAML规则文件，可直接用于patsona的专利分类流程。

---

## When to Use

**明确构建意图：**
- 用户说"构建分类体系"、"创建分类规则"、"新建技术分支"
- 用户说"分类体系构建"、"patent taxonomy"
- 用户需要为某个技术方向建立专利分类标准
- 用户需要定义专利分类的层级结构和判定规则

**前置条件：**
- 用户有明确的或模糊的技术方向需求
- patsona项目可访问（用于写入rules目录）

---

## 工作流程

### Step 1: 确定技术方向 `[Step 1/5]`

**AI行为：**
1. 用 AskUserQuestion 询问技术方向名称
2. 基于技术常识推荐该方向常见的一级分支供用户选择
3. 用户可自由输入或从推荐中选择

**AskUserQuestion 示例：**
```
问题1: 技术方向名称？（如：电池、半导体、人工智能）
问题2: 以下推荐的一级分支是否合适？可多选或自定义补充
  - [AI根据方向推荐3-5个一级分支]
```

**输出：** 技术方向名称 + 一级分支列表

---

### Step 2: 逐层构建分支树 `[Step 2/5]`

**AI行为：**
1. 对每个一级分支，询问其下的二级细分
2. 对每个二级分支，询问其下的三级细分
3. AI根据已有分支智能推荐可能的细分方向
4. 每层用 AskUserQuestion，选项包含AI推荐 + 自定义输入
5. 构建过程中实时展示当前分类树结构

**实时展示格式：**
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🌲 当前分类树
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
电池
├── 锂离子电池
│   ├── 磷酸铁锂
│   ├── 三元材料
│   └── [待补充]
├── 固态电池
│   └── [待补充]
└── [待补充]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**输出：** 完整的三级（或四级）分支树

---

### Step 3: 定义末端分支判定标准 `[Step 3/5]`

**AI行为：**
1. 对每个最细粒度分支，引导用户描述：
   - 关键区分特征（"什么条件下归入这个分支"）
   - 关键词/正则规则（用于Layer1粗筛快速排除）
   - 边界案例处理规则（"如果同时涉及A和B怎么分"）
2. AI根据分支名称和上下文推荐初始判定标准
3. 用户确认或修改

**AskUserQuestion 示例：**
```
问题1: "磷酸铁锂"的关键区分特征？
  - AI推荐: 包含LFP/磷酸铁锂/LiFePO4等关键词
问题2: 边界案例：同时涉及磷酸铁锂和三元材料时如何归类？
  - 优先归入主权利要求所述材料
  - 按创新点所属材料分类
  - 其他
```

**输出：** 每个末端分支的判定标准

---

### Step 4: 导入已标注样本 `[Step 4/5]`

**AI行为：**
1. 询问用户是否有已标注的专利样本
2. 如有，确认Excel文件路径（.xlsx格式）
3. 解析Excel后，AI将样本自动匹配到对应分支
4. 对未匹配的样本，提示用户确认分类

**Excel预期列：** 专利号、标题、摘要、分类标签

**输出：** 样本匹配结果 + 未匹配样本列表

---

### Step 5: 生成规则文件并校验 `[Step 5/5]`

**AI行为：**
1. 自动生成YAML规则文件到 `patsona/rules/{技术方向}/` 目录
2. 校验完整性：
   - 是否有分支无判定标准
   - 是否有关键词重叠导致模糊
3. 展示最终分类树 + 规则统计
4. 用 AskUserQuestion 确认是否满意

**校验报告格式：**
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 规则校验报告
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
分支总数: 15
有判定标准的分支: 14
⚠️ 缺少判定标准: [固态电池-氧化物电解质]
关键词重叠: [锂离子电池/固态电池 共享关键词"锂"]
样本覆盖: 45/50 (90%)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**输出：** YAML规则文件 + 校验报告

## 硬规则

### P0 - 必须遵守

- **必须使用 AskUserQuestion 工具进行交互**，每次最多4个问题
- **禁止跳过任何步骤**
- **禁止假设用户同意**（必须明确确认）
- **禁止生成无判定标准的分支**
- **每轮对话必须显示进度**：`[Step X/5]`

### P1 - 重要规则

- AI应主动推荐（基于技术常识），用户只需确认或微调
- 增量构建：不需要一次建完所有方向
- 实时展示当前分类树，让用户始终看到全局
- 样本导入后必须验证规则覆盖度

### P2 - 推荐规则

- 推荐三级深度（最多四级），避免过度细分
- 推荐每个分支至少3个关键词
- 推荐对边界案例提供明确处理规则
